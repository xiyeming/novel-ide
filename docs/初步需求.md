# Novel IDE 产品需求文档（PRD）

# 1. 产品定位

Novel IDE 是一款基于 **Rust 1.90+ + Tauri 2 + Vue3/React（任选）** 构建的专业小说创作 IDE，面向长篇小说作者、网文作者、编剧及 AI 辅助创作者。

产品定位不是聊天工具，而是一个完整的创作生产平台，融合：

* AI Agent
* MCP（Model Context Protocol）
* Skills
* Hooks
* Workflow
* RAG
* 项目管理
* 角色管理
* 世界观管理
* 长上下文创作

核心目标：

> 让 AI 成为真正的小说创作团队，而不是聊天机器人。

# 21. 非功能需求

性能目标：

* 冷启动：≤ 2 秒（SSD 环境）
* 项目打开：≤ 3 秒（10 万章节索引以内）
* 单章生成：支持流式输出，首 Token 响应 ≤ 1 秒（取决于模型）
* RAG 检索：≤ 200 ms（百万字知识库）
* 全文搜索：≤ 100 ms（SQLite FTS5）
* SQLite 数据库容量：≥ 10 GB
* 支持百万字以上长篇小说持续创作
* 离线优先，网络异常时本地功能可正常工作

## 建议增加的高级能力（推荐纳入 V1.5/V2）

基于你的设计，再补充一些能够明显提升专业性的功能：

1. **Story Memory（故事长期记忆）**：自动生成章节摘要、角色状态快照、世界观变化记录，减少长篇创作时上下文丢失。
2. **Timeline（时间线）**：事件、角色、地点统一时间轴，可检测时间冲突。
3. **Plot Graph（剧情图）**：伏笔、冲突、高潮、反转、角色关系可视化。
4. **Version Snapshot（版本快照）**：章节和项目支持 Git 风格历史、差异比较与一键回滚。
5. **Prompt 调试器**：查看最终拼接 Prompt、Token 占用、RAG 命中、Agent 调用链。
6. **Workflow Designer**：提供可视化节点编排（类似 Node-RED / LangFlow），自定义小说创作流水线。
7. **Plugin SDK**：允许第三方开发 Agent、Skill、Hook、导入导出器、编辑器扩展、主题等插件。
8. **事件总线（Event Bus）**：统一 `ProjectOpened`、`ChapterGenerated`、`ReviewCompleted` 等事件，供 Hook、插件和 UI 响应，实现松耦合扩展。

整体来看，这将是一款真正意义上的 **AI Native Novel IDE**，其定位更接近 **VS Code + JetBrains + Obsidian + Scrivener + Claude Code + Cursor** 的融合，而不是传统的 AI 写作助手。
